owocr / api

HTTP control interface

default url http://localhost:7332
All endpoints accept POST requests. Responses are plain UTF-8 text. On success the status is 200; on failure 400 with an error message in the body. Endpoints that require a body expect it as text/plain with a correct Content-Length header. Maximum body size is 65 536 bytes.
endpoints
POST /action/hello health check
Description Sanity check. Returns a greeting if the server is up.
Request body none
Example 
curl -X POST http://localhost:7332/action/hello
Responses
200 OK
Hello, world!
POST /action/capture trigger screenshot
Description Queues a screenshot capture request.
Request body none
Example 
curl -X POST http://localhost:7332/action/capture
Responses
200 OK
Capture triggered
POST /action/pause toggle pause
Description Toggles the paused state of the OCR loop.
Request body none
Example 
curl -X POST http://localhost:7332/action/pause
Responses
200 OK
Paused
POST /action/change-engine switch OCR engine
Description Switches the active OCR engine by index.
Request body
text/plain — integer
Zero-based index of the engine to activate. Must be within the range of configured engine instances.
Example 
echo "0" | curl -X POST http://localhost:7332/action/change-engine \
  -H "Content-Type: text/plain" -d @-
Responses
200 OK
Engine changed to <name>
400 Bad Request
body is empty
invalid engine index
engine index out of range
invalid content-length
body too large
truncated body
